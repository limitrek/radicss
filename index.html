Radicss

A simple CSS framework.

Containers

Radicss utilizes two classes to wrap content: .container and .container-fluid. The former has a maximum width set for each of the breakpoints we have defined. The later utilizes 100% of the available width in all situations.

The following breakpoints have been defined in Radicss:

  • sm: 576px
  • md: 768px
  • lg: 992px
  • xl: 1200px
  • x2: 1400px (add .x2 to your container to enable this size)
  • x3: 1600px (add .x3 to your container to enable this size)
  • x4: 1800px (add .x4 to your container to enable this size)
  • x5: 2500px (add .x5 to your container to enable this size)

Grid system

Radicss has two grids, one with 12 cols and other with 15 cols. To use them, place an element with the .row class inside a container and you will be able to put columns inside it using the .col-* classes.

The column classes can be read as follows:

.col-{useThisManySpaces}-{outOfThis}

Responsive breakpoints

In the previous section a col always has the same width no matter which screen size is being used. To add responsiveness to the grid system, use a column class with a breakpoint, such as .col-sm-6-12.

The responsive column classes can be read as follows:

.col-{atThisScreenSizeAndAbove}-{useThisManySpaces}-{outOfThis}

For example, the following class:

col-12-12 col-sm-6-12 col-md-4-12 col-lg-3-12

Can be read as follows:

  • col uses 12 out of 12 at the smallest screen size
  • At sm size (576px), col uses 6 out of 12
  • At md size (768px), col uses 4 out of 12
  • At lg size (992px), col uses 3 out of 12
  • Although the xl size is not stated, the col will use 3 out of 12
